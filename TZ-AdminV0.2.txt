Дополнитекущий проет.
Создай приложение для администратора. Предполагается использовать его для заполнения расписания, в будущем функционал будет расширяться.

Стек:
1. Backend: NestJS + TypeScript + MongoDB (Mongoose)
2. Frontend: React или Next + TypeScript, использую Ant Design, современный дизайн и адаптивную верстку.
3. База данных: MongoDB
4. Использую Docker.

При входе добавть форму с аутентификацией.

Модель данных:
Событие дня содержит следующие поля:
1. id - уникальный идентификатор;
2. date - обязательное поле. Указывает на дату планируемого события. Возможность выбрать цвет при отображении фронтенда
3. title - обязательное поле, от 3 до 100 символов;  Возможность выбрать цвет при отображении фронтенда.
3. additional_title - необязательное поле, от 3 до 100 символов;  Возможность выбрать цвет при отображении фронтенда.
4. description - необязательное поле, до 500 символов;  Возможность выбрать цвет при отображении фронтенда
5. communicants - необязательное поле, сожержит число
6. parishioners - необязательное поле, сожержит число
7. other - необязательное поле, до 1000 символов;
8. status - одно из значений: new, in_progress, agreed, done;
9. priority - одно из значений: low, normal, high;
10. created_at - дата и время создания в UTC
11. updated_at - дата и время последнего изменения в UTC

Функциональные требования
Реализуй:
1. Создание события.
2. Просмотр события согласно календарю.
3. Фильтрацию по status и priority .
4. Поиск по date, title .
5. Сортировку по дате создания и статусу.
6. Возможность редактирования события.
7. Удаление события.
8. Пагинацию списка.

Бизнес-правила: 

1. Аккаунт админский с дефолтными кредами (admin:admin), в будущем будет изменено.
3. При нарушении правила API должен возвращать осмысленный HTTP-ответ и понятное сообщение об ошибке.
5. Поиск, фильтрация, сортировка и пагинация должны выполняться на backend, а не только на frontend.

Требования к frontend
Сделай один экран приложения, включающий:
1. таблицу или список событий;
2. строку поиска;
3. фильтры по статусу;
4. управление сортировкой;

При заполнении информации каждого события пусть внезу появляется небольшое окно с сайта с соответствующей датой, на пример: https://www.patriarchia.ru/bu/2026-07-02

1. выбор цвета. Должен быть как пользовательский набор цветов (черный, темно-жетлый, голубой или синий, фиолетовый, зеленый, красный, бордовый), так и небольшое поле с возможностью выбрать свой цвет.
2. Отображение по календарю согласно неделям. Первый день понедельник, последний воскресенье.
